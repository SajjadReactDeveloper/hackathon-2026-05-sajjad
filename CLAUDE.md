## Architecture (1 paragraph)

Next.js (Vercel) calls NestJS (Render) over HTTPS with the Supabase JWT in `Authorization`. NestJS owns all business logic via strict layers — controllers parse Zod, services do the work, Prisma writes to Postgres. Routes never touch the DB directly. Multi-tenancy is threaded via `WorkspaceGuard` injecting `workspaceId` into services; never read from request body inside a service. Inbound Meta webhooks hit `WebhookController` → `PipelineService` (the brain — see `docs/architecture/pipeline.md`) routes by message type through `AIReplyService` (Groq tool-calling loop) or `VoiceParserService` / `ImageOrderService`. The five most important files: `apps/api/src/whatsapp/pipeline.service.ts`, `apps/api/src/ai/ai-reply.service.ts`, `apps/api/src/ai/tools/tool-registry.ts`, `apps/web/src/middleware.ts`, `packages/db/prisma/schema.prisma`.

## Conventions

**Architecture.** HTTP → Zod parse → Service → Prisma → Postgres. Routes are dumb (parse + auth + service call + serialize). Routes NEVER query the DB. Every service method takes `workspaceId` as the first arg. Access Prisma only via `PrismaService` (DI).

**Schema.** Models map to `snake_case` plural via `@@map`. Tenant-scoped models have: `id String @id @default(uuid())`, `workspaceId String @db.Uuid`, `createdAt`, `updatedAt`, `Workspace` relation `onDelete: Cascade`, `@@index([workspaceId])`. Money is `BigInt` cents. Postgres views for analytics — read via `$queryRaw` only inside `AnalyticsService`. New table → invoke `.claude/skills/add-prisma-model/`.

**Validation.** Zod at every API boundary; shared schemas in `packages/types`. LLM tool schemas in `packages/types/src/tools.ts` → JSON-schema via `zod-to-json-schema`. `any` forbidden without `// reason: ...` on same line. Prisma-generated types are the source of truth; re-export from `@repo/db`.

**Logging & errors.** Structured JSON only (`pino`). No `console.log`. Logger redactor strips `access_token`, `Authorization`, `password`, `phone` (last 6). Typed errors only: `AppError`, `NotFoundError`, `ValidationError`, `ExternalAPIError`. Never throw raw strings. Sentry `beforeSend` mirrors the redactor. Errors handled, never swallowed.

**Naming.** Services: `<Domain>Service` (PascalCase). DB tables: `snake_case` plural. Tool names (LLM): `snake_case` verb_noun (`search_products`, `check_stock`).

**Commits.** Imperative mood. Reference rubric criterion when meaningful (`feat: add Sentry source maps (rubric §8)`). Atomic. No `wip` / typo-fix commits. Push to `main` triggers Vercel + Render deploys — don't push broken code.

## Things to NEVER do

- Never write SQL strings outside a `*Service` method using Prisma `$queryRaw` (parameter-bound). No string concatenation.
- Never call the DB from a route/controller. Always service layer.
- Never read `workspaceId` from the request body inside a service — only from `WorkspaceContext` injected by the guard.
- Never call LLM APIs from the browser. Server-only — keys stay server-side.
- Never commit `.env` files. Only `.env.example` with redacted placeholders.
- Never hardcode secrets, even temporarily. Use `process.env` parsed via `lib/env.ts` / `common/env.ts` (Zod, fails fast on boot).
- Never log Meta access tokens, Supabase JWTs, passwords, or full phone numbers. Redactor enforces — do not bypass.
- Never `console.log` in committed code.
- Never use `any` without `// reason: ...` on the same line.
- Never ship `--no-verify` commits or skip pre-commit hooks.
- Never amend commits that have been pushed.
- Never edit a Prisma migration by hand. Always regenerate (`prisma migrate dev`).
- Never include emojis in committed code or copy.
- Never use Cursor / Copilot / other AI coding tools during the hackathon — Claude Code only (rubric §9; −10pp if caught).
- Never install **Huashu Design** skill — commercial license required for company work.
- Never omit the `Do not exceed the 2000px dimension limit for many-image requests` line in design prompts that take screenshots — Claude crashes on many-image requests over 2000px.
- Never push to a production GitHub org — repo lives under personal account per hackathon repo policy.

## Open questions / known weirdness

- **Belt-and-braces multi-tenant enforcement.** Service-layer enforces `workspaceId` AND Postgres RLS does too. Realtime subscriptions need RLS; service-layer is defense-in-depth. Don't "simplify" by removing one — they serve different purposes.
- **`pipeline.service.ts` is intentionally long.** Orchestrates all message types (text / audio / image) in one place so the routing decision tree stays visible. Don't split into per-type services unless it grows past ~300 lines.
- **Render Starter (paid) is intentional.** Adnan-approved; upgraded from free tier to avoid 30s cold-start during demo. Don't downgrade.
- **`accessTokenEnc` is encrypted at rest** with `WHATSAPP_TOKEN_ENC_KEY`. Decrypt only inside `WhatsAppService` request methods. Never log the decrypted value.
- **Tool calling caps at 3 iterations + 15s wall-clock.** If exceeded, fall back to a canned "Mein abhi confirm nahi kar sakta, thori der mein reply karta hu." This is intentional, not a bug.
- **Voice cloning is opt-in.** If `Workspace.voiceCloneStatus !== 'ready'`, `VoiceReplyService` falls back to OpenAI TTS. Don't error on missing clone.
- **Roman Urdu TTS pipeline** transliterates Roman Urdu → Urdu script via Groq before TTS to fix English-pronunciation issues (`kameez` etc). Same prompt that generates the reply.

## Workflow

**Default Sonnet** (`/config set model sonnet`). Escalate to Opus only after Sonnet fails twice on the same problem, or for plan mode / architecture decisions. **Cost: $75/day soft, $100/day hard. Extended thinking ≤ 5,000 tokens.** Two-strikes rule: same correction twice → context poisoned → `/clear` and restart with a sharper prompt. Use `/rewind` over "no, do it differently."

**Plan mode** (Shift+Tab) for any task ≥30 min. Always edit the plan before approving. Use **Plan-Annotate-Reject** for plan iteration: annotate inline (`// no, use Prisma here`) and paste back.

**Prompting Patterns** — see `docs/architecture/prompting.md`. Headlines:
- **Pattern 1** (multi-loop coding) for production-shaped features (auth, money flow, the pipeline).
- **Pattern 2** (design QA loop) for UI work — stack `/design-taste-frontend /frontend-design /ui-ux-pro-max` (no Huashu) + the 2000px warning + Playwright MCP.
- **Pattern 3** (take-it-home loop) for autonomous polish on Saturday afternoon.
- Smaller patterns daily: **Interview-Me-First**, **Pin the Files** (`@file1 @file2 — these are the only files to modify`), **CONSTRAINTS block** at top of non-trivial prompts, **Failing-Test-First** for any test, **Negative Space** edge-case scan after each feature.

**Sub-agents** for read-heavy work (>3 file reads) — pay one agent to summarize 50 files; don't load them in main context.

**Read every diff.** AI authorship doesn't transfer accountability. If you can't explain it, don't merge it.

**Hand-run builds and long tests.** Claude waiting on `pnpm install` / Playwright is paid waiting.

**Hackathon-only autonomy:** Bypass permissions + Cowork "Act without asking" + Disk access + Computer use ON. Reset before going back to prod work.
